## 1. Partes y objeto

Estos términos regulan la contratación del software de gestión de conserjería **Portia**, provisto
por **[razón social / RUT — COMPLETAR]** ("Portia"), por parte de una administradora de edificios
o comunidad ("el Cliente").

Portia es un servicio de software como servicio (SaaS) compuesto por:
- una aplicación de escritorio para conserjes, y
- un panel web para administradores,

que digitalizan el registro de visitas, encomiendas, novedades, tareas y turnos del edificio.

## 2. Cuenta y acceso

- El Cliente designa a uno o más administradores, que a su vez dan de alta a los conserjes.
- El Cliente es responsable de la veracidad de los datos de sus usuarios y del uso que hagan de
  sus credenciales.
- Portia puede suspender el acceso ante un uso que comprometa la seguridad del servicio o de
  terceros.

## 3. Protección de datos personales

- El Cliente actúa como **responsable** del tratamiento de los datos personales de visitantes y
  residentes; Portia actúa como **encargado** que los procesa por cuenta del Cliente.
- Las condiciones de este tratamiento se rigen por el **Acuerdo de Tratamiento de Datos (DPA)**
  ([`DPA-acuerdo-tratamiento-datos.md`](DPA-acuerdo-tratamiento-datos.md)), que forma parte
  integrante de estos términos.
- El Cliente se obliga a tener una base legal para registrar los datos de las personas (por
  ejemplo, consentimiento o interés legítimo de seguridad) y a informar a los titulares conforme
  a la **Ley 21.719**.

## 4. Disponibilidad del servicio

- **[Pre-alpha: sin SLA comprometido.]** Durante la etapa de validación temprana, el servicio se
  ofrece "tal cual" (*as is*), sin garantía de disponibilidad ni de continuidad.
- Cuando exista un plan comercial, el nivel de servicio (SLA) se definirá en el plan contratado.
- Portia puede realizar mantenimientos que interrumpan temporalmente el servicio, avisando con
  razonable anticipación cuando sea posible.

## 5. Modo offline y sincronización

- La aplicación de escritorio permite registrar información sin conexión, que se sincroniza al
  reconectar. El Cliente entiende que, mientras un dispositivo está sin conexión, los datos aún
  no sincronizados solo existen localmente en ese dispositivo.
